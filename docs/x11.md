# X11 Support

StarryOS supports X11 for GUI.

## Steps

1. Make sure you run StarryOS with necessary flags:

   ```bash
   make img ARCH=riscv64
   make run ARCH=riscv64 GRAPHIC=y INPUT=y
   ```

   For the lab workflow, there is also a prepared manual entry point:

   ```bash
   make run-x11
   ```

   After the first staged run, use the cached image for faster relaunches:

   ```bash
   make justrun-x11
   ```

2. In StarryOS, install necessary dependencies:

   ```bash
   apk add xorg-server xf86-video-fbdev xf86-input-evdev
   ```

   If you used `make run-x11`, the X11 userspace is already staged into the
   guest image and you can skip the `apk add` step.

3. Start the X server and the teaching calculator client:

   ```bash
   /usr/bin/lab_x11demo
   ```

   The helper starts `X -retro` and runs the built-in Starry Lab teaching
   calculator in the foreground. Close the calculator to get the shell back.

4. If you want to split server and client manually:

   ```bash
   /usr/bin/lab_x11demo server
   export DISPLAY=:0
   /usr/bin/lab_xcalc
   ```

5. `dwm` is also available as a lightweight window manager.

   ```bash
   apk add dwm
   dwm &
   ```
